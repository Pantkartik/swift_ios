# Protocols in Swift 📜

Protocols define a blueprint of methods, properties, and other requirements that suit a particular task or piece of functionality. The protocol can then be adopted by a class, structure, or enumeration to provide an actual implementation of those requirements.

## 📄 Contents

- `Protocols_Basics.swift`: Introduction to protocols, protocol syntax, property requirements, and method requirements.
- `Protocols_Advanced.swift`: Advanced concepts like protocol extensions, opaque return types (`some Protocol`), and primary associated types (`any Protocol`).

## 💡 Key Concepts
- **Protocol Extensions**: Allow you to provide default implementations for protocol methods.
- **Associated Types**: Give a placeholder name to a type that is used as part of the protocol.
- **Opaque vs Existential Types**: Understanding the difference between `some` and `any` keywords in Swift 5.7+.
